# Tools

This folder is dedicated to the tools used in modern C++ development.

The goal of this section is to introduce the tools that every C++ developer should know when starting a new project. Instead of covering every tool that has ever existed, this repository focuses on modern, actively maintained, and industry-standard tools that are widely adopted in professional C++ development.

Each topic includes practical guidance, best practices, and references to the tool's official documentation so you can learn directly from the source while following real-world workflows.

Topics include, but are not limited to:

* Debuggers
* Memory Profilers and Memory Tracers
* Testing Frameworks
* Documentation Generators
* Static Analyzers
* Performance Profilers
* Build Systems
* Package Managers
* Code Formatters and Linters
* Sanitizers

Each guide covers:

* What the tool is and why it exists.
* When and where to use it.
* Installation and setup.
* Practical workflows and examples.
* Best practices.
* Common pitfalls and how to avoid them.
* References to the official documentation and other authoritative resources.

The philosophy of this repository is simple: if you're starting a new C++ project today, you should learn the same tools and workflows used by modern C++ developers in production. Whenever possible, recommendations prioritize actively maintained, industry-standard solutions over outdated or legacy alternatives, with official documentation serving as the primary reference.
